# 小程序与多端模式限制清单

本项目当前使用微信开发者工具 `multiPlatform` 项目结构，可以继续输出微信小程序和多端 App。这个模式解决的是多端产物组织问题，不会取消微信小程序运行时和发布审核的限制。

## 小程序端必须遵守的限制

1. 包体限制
   - 主包和分包仍受微信小程序包体限制。
   - 生成图、趋势图、用户图不能长期打进主包。
   - 当前开发阶段只允许保留压缩测试 JPG，原始大图必须通过 `packOptions.ignore` 排除。
   - 正式环境生成图应上传到 OSS / COS / S3 / CDN，通过远程 URL 展示。

2. 网络限制
   - 小程序正式环境必须使用已配置的合法 HTTPS 域名。
   - 不能在正式环境依赖 `127.0.0.1`、`localhost`、裸 IP 或未备案域名。
   - moxing API Key 只能保存在后端环境变量中，小程序不直连 moxing。
   - 小程序只调用我们的业务后端，后端负责提交、轮询、下载和失败重试。

3. 异步任务限制
   - 图像生成不能设计成前端长时间等待一个请求返回。
   - 前端只创建任务、查看任务状态、轮询结果。
   - 轮询间隔建议 2-5 秒，单页同时轮询不超过 1-2 个任务。
   - 状态统一使用 `pending / submitted / queued / running / ready / failed / incomplete`。

4. 权限和隐私
   - 三视图、自拍、衣橱图、生成图保存都属于敏感图片链路。
   - 正式版必须补齐隐私协议、相册/相机授权说明、删除入口。
   - 用户图片默认不公开展示，不用于未经授权的训练和营销素材。

5. UI 与配置限制
   - 原生 `tabBar` 最多 5 项，当前已经用满：今日、建档、点评、衣橱、我的。
   - 详情页如果需要底部导航，应继续使用自定义底栏，不加入原生 tabBar。
   - 首页图片、Logo、主题切换、场景切换要按小屏真机优先验证。

## 多端 App 需要单独处理

1. `project.miniapp.json` 是多端 App 输出配置入口。
2. Android / iOS 需要补齐图标、启动图、隐私开关、证书和包名配置。
3. 不要把核心业务绑定到微信专属能力；登录、建档、生成、衣橱、详情都应走统一后端 API。
4. 多端 App 可以承载更多本地资源，但生产图仍建议走远程对象存储，避免包体膨胀和热更新困难。
5. 多端真机验证必须拆成三组：微信开发工具预览、微信小程序真机、Android / iOS App 真机。

## 当前工程执行规则

- 保持 `projectArchitecture: "multiPlatform"`。
- 保持 `backend`、`docs`、`prototypes`、原始大图在小程序预览包之外。
- 开发环境允许本地后端 `http://127.0.0.1:8787`。
- 正式联调前必须准备 HTTPS 后端域名和远程图片域名。
- 生图服务固定通过后端 `image-jobs` 协议，不允许前端保存第三方模型 Key。
