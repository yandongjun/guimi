# 《闺蜜》AI穿搭产品说明书（AI开发版 V1.2）

## 1. 产品基本信息

### 产品名称

- 小程序 / 产品名：闺蜜
- AI角色名：米粒（Mili）
- 视觉字标：MILI + 竖排“闺蜜”

### 产品定位

闺蜜是一款面向女性用户的 AI 穿搭陪伴小程序。它不是单纯的 AI 换装工具，也不是高高在上的时尚编辑，而是一个“审美在线、说话具体、不制造身材焦虑”的日常穿搭搭档。

系统通过用户三视图、身材比例、审美偏好、用户衣橱、当地天气、使用场景、流行趋势和 AI 图像生成能力，为用户每天提供一套“适合自己、能现实穿、且符合当下审美”的穿搭方案。

核心目标：

> 让用户每天打开闺蜜，都能看到一张自己穿上今日推荐衣服的效果图，并知道为什么这样穿。

## 2. 产品核心判断

### 产品不是

- 不是普通 AI 换装工具。
- 不是只给文字建议的穿搭助手。
- 不是只做商品导购的带货平台。
- 不是评价用户身材的审美裁判。

### 产品是

- 长期陪伴型 AI 时尚闺蜜。
- 能记住用户身材、衣橱、偏好和生活场景的穿搭顾问。
- 能把建议可视化成试穿效果图的审美工具。
- 能帮助用户逐步形成个人衣橱和个人风格的成长型产品。

### 第一性体验

用户打开首页时，最重要的不是看到一段建议，而是看到：

1. 自己穿上今日推荐搭配后的效果图。
2. 图上有简洁的穿搭结论和关键理由。
3. 天气、幸运色、场景、比例策略都服务于这张图。
4. 用户可以快速切换场景，让米粒重新搭一套。

## 3. 目标用户

### 核心用户

- 18-38 岁女性。
- 日常有穿搭需求，但不想每天花太多时间搭配。
- 希望自己更好看，但不喜欢被评价身材。
- 愿意上传照片换取更个性化的建议。
- 对小红书、韩系、通勤、清爽、轻熟、干净感等风格敏感。

### 高频场景

- 上班
- 约会
- 聚会
- 出游
- 逛街
- 婚礼 / 正式场合
- 校园
- 音乐节 / 展览 / 咖啡

### 用户心理

用户真正想要的不是“系统告诉她穿什么”，而是：

- 今天不用纠结。
- 这套穿出去不会出错。
- 这套确实适合我的身材。
- 如果天气变了，我也知道怎么脱外套、怎么补层次。
- 米粒懂我，而且越用越懂我。

## 4. 米粒人设

### 角色定位

米粒是用户的 AI 闺蜜穿搭搭子。她不需要特别漂亮，不制造压迫感，但必须显得懂时尚、懂生活、懂用户。

### 形象方向

- 亲近、普通、干净、有审美。
- 不做强势美女人设。
- 不做冷冰冰专家。
- 不做夸张虚拟偶像。

### 语气原则

- 少空话，多理由。
- 不说“你不适合”，改说“这类版型对你不划算，可以换成……”
- 不制造身材羞耻。
- 评价必须有证据：比例、颜色、场景、天气、衣橱单品。
- 每次指出问题时，必须给替代方案。

### 示例语气

```text
雾蓝今天点到为止就好，放在内搭或配饰上会更轻。
外套可以短一点，肩线会更清楚，腰线也能自然往上走。
今天昼夜温差大，外套要能脱，内搭单穿也要完整。
```

## 6. 首页产品逻辑

### 首页核心内容

首页首屏必须围绕“今日试穿效果图”组织，而不是文字卡片。

首屏包含：

- 日期与当地天气。
- 星座幸运色或今日颜色提示。
- 页面主题切换：米色、粉色、酷炫。
- 今日穿搭标题。
- 用户穿上推荐搭配后的 AI 效果图。
- 图上的关键标注：补肩线、高腰线、比例、颜色、场景。
- 底部品牌字标：竖排“闺蜜” + MILI。

首屏下方包含：

- 场景切换：上班、约会、聚会、出游、更多。
- 切换场景后重新生成或重新推荐穿搭。

### 首页文案长度约束

首页是图片优先的杂志封面式界面，AI 每天输出的文字必须短，不能把页面挤乱。

建议限制：

| 文案位置 | 字数限制 | 示例 |
| --- | --- | --- |
| 今日穿搭标题 | 6-10 个中文字符 | 薄外套 + 清爽内搭 |
| 今日情绪说明 | 8-14 个中文字符 | 温柔但有边界感 |
| 天气胶囊 | 12 个中文字符以内 | 上海 27/18°C |
| 幸运色胶囊 | 12 个中文字符以内 | 天秤幸运色：雾蓝 |
| 主题胶囊 | 4-6 个中文字符 | 主题 米色 |
| 图上标签 | 2-4 个中文字符 | 补肩线 / 高腰线 |
| 理由标题 | 2 个中文字符 | 比例 / 颜色 / 场景 |
| 理由短句 | 4-8 个中文字符 | 优化肩胸比 |

如果 AI 生成的文案超过限制，前端不应直接展示长文案，而应使用压缩后的 `display_text` 字段。

推荐接口返回：

```json
{
  "title": "薄外套 + 清爽内搭",
  "display_title": "薄外套 + 清爽内搭",
  "mood": "今天走温柔但有边界感",
  "display_mood": "温柔但有边界感"
}
```

### 首页推荐公式

```text
今日推荐 =
当地天气
+ 昼夜温差
+ 用户身材比例
+ 用户衣橱已有单品
+ 用户偏好颜色
+ 星座幸运色
+ 使用场景
+ 最新趋势资产
```

### 天气策略

天气不是展示信息，而是穿搭决策依据。

示例：

- 昼夜温差大：推荐可脱外套，且内搭单穿完整。
- 高温：减少层次，强调透气材质。
- 下雨：避免拖地裤、浅色麂皮、难打理鞋包。
- 大风：避免太轻飘的裙摆，或提醒加固定层。

### 幸运色策略

幸运色是情绪价值，不应该压过穿搭合理性。

规则：

- 可作为点缀色、内搭色、配饰色。
- 不强制大面积使用。
- 与用户偏好冲突时，以用户偏好和整体效果优先。

## 8. 三视图与身材分析系统

### 建档页定位

建档页不应只是“上传三视图”的工具页，而应是“米粒了解你”的入口。

第一阶段建档页包含：

- 快速选项：年龄段、星座、城市、身高、体重、常用场景。
- 三视图上传：正面、侧面、背面。
- 身材比例分析：三张照片齐全后生成比例档案。

第一阶段建档不放大段自由输入框，避免用户不知道该写什么。自然语言采集放到全局米粒聊天里，在用户有具体需求时再收集。

建档完成条件：

```text
年龄段 + 星座 + 城市 + 身高 + 体重 + 常用场景 + 三视图
```

建档结构化字段：

```json
{
  "age_range": "23-28",
  "zodiac": "天秤座",
  "city": "上海",
  "height_cm": 164,
  "weight_kg": 50,
  "scene_frequency": ["office", "shopping"]
}
```

### 上传内容

用户上传：

- 正面照
- 侧面照
- 背面照

### 上传要求

- 全身入镜。
- 光线正常。
- 背景简单。
- 穿贴身或能看清轮廓的衣物。
- 不强制暴露隐私部位。

### 分析目标

系统输出：

- 身材比例。
- 骨架与肩线判断。
- 腰臀比例。
- 腿长比例。
- 头身比。
- 适合版型策略。
- 需要规避的版型风险。

### 标签体系优化

女生衣服只靠“风格标签”无法筛出真正适合用户的衣服。标签体系必须服务推荐决策，而不只是描述衣服看起来属于什么风格。

推荐采用多维标签：

| 标签维度 | 示例 | 用途 |
| --- | --- | --- |
| 品类 | top、outerwear、pants、dress | 判断搭配结构 |
| 子品类 | blazer、shirt、camisole、wide_leg_pants | 精确召回单品 |
| 版型轮廓 | short_jacket、long_coat、a_line、straight、wide_leg | 判断身材适配 |
| 合身度 | slim、regular、oversize、cropped、high_waist | 判断比例效果 |
| 颜色 | mist_blue、cream_white、denim_blue | 匹配幸运色和偏好 |
| 色彩角色 | main_color、accent_color、neutral_base | 判断颜色占比 |
| 材质 | cotton、linen、knit、satin、denim | 判断季节和质感 |
| 厚薄保暖 | light、medium、warm、windproof | 匹配天气 |
| 正式度 | casual、smart_casual、formal | 匹配场景 |
| 风格 | korean、clean_fit、minimal、french | 匹配审美偏好 |
| 身材策略 | raise_waistline、extend_leg、define_shoulder、soften_hip | 判断是否适合用户 |
| 风险标签 | press_height、widen_shoulder、expose_hip、too_mature | 避免不适合推荐 |
| 搭配关系 | good_with_tank、good_with_wide_pants、needs_inner_layer | 生成完整搭配 |

### 关键结论

推荐系统不应只问“这件衣服是什么风格”，而应能回答：

```text
这件衣服适合谁？
适合什么天气？
适合什么场景？
能解决用户哪个比例问题？
和用户衣橱里的哪些单品能组成完整搭配？
如果用户衣橱没有，是否需要从趋势库找替代？
```

## 11. AI穿搭推荐系统

### 推荐目标

根据用户身材、喜好、衣橱、趋势、天气和场景，生成一套现实可穿的搭配方案。

### 推荐优先级

1. 优先使用用户已有衣橱。
2. 如果用户衣橱无法满足目标效果，再从趋势库或商品候选库中寻找替代。
3. 如果用户没有衣橱，默认从趋势库中生成可参考穿搭。
4. 场景和天气必须先满足。
5. 风格和幸运色作为加分项。
6. 推荐必须能解释。
7. 用户没有主动询问购买渠道时，不主动展示购买链接。

### 单品来源策略

推荐系统需要区分三类单品来源：

| 来源 | 说明 | 展示策略 |
| --- | --- | --- |
| 用户衣橱 | 用户已上传或识别出的真实单品 | 优先使用，直接推荐 |
| 趋势库 | 后台维护的流行穿搭参考 | 可用于生成灵感和替代方案 |
| 商品候选库 | 带有购买链接的可购买单品 | 只在用户需要时展示 |

当用户衣橱能满足效果时，不引导购买。

当用户衣橱不能满足效果时，米粒可以说：

```text
你衣橱里暂时没有类似的短外套，我先用趋势库里的同类版型帮你试一版。
```

当用户主动问“哪里可以买”或点击“找同款/类似款”时，再展示购买链接。

同类型衣服不只推荐一个链接，必须提供多个候选，避免用户误以为系统只是在推某一个商品。

推荐同类商品时，每个候选至少包含：

- 商品图
- 来源平台
- 价格区间
- 颜色
- 关键标签
- 为什么像这件
- 适合用户的理由

## 12. AI图片生成系统

### 生成引擎

调用：

```text
Image2 API
```

### 生成输入

系统需传入：

- 用户参考图 URL：三视图、自拍或历史试穿图。
- 穿搭参考图 URL：用户衣橱单品、趋势图、商品图。
- 身材比例结构化数据。
- 场景、天气、风格和颜色要求。
- Prompt 模板。

### 图片生成原则

- 生成的是“用户穿上推荐搭配后的效果图”。
- 不过度改变用户身材。
- 不过度美颜。
- 不生成不现实的版型和材质。
- 背景应服务穿搭，不抢主体。
- 首页图优先使用 PNG 透明人物图或可换背景图，便于主题切换。

### Prompt 方向

必须强调：

```text
young
trendy
2026 fashion
xiaohongshu style
social media fashion
fashion editorial
asian fashion
korean fashion
natural beauty
daily outfit
street photography
casual chic
daily commuting
natural pose
```

避免：

```text
mature styling
wedding studio style
heavy makeup
old fashioned
luxury studio portrait
over-retouched skin
unrealistic body proportions
```

## 13. 穿搭评价系统

### 功能说明

用户上传穿搭照片后，系统给出有理有据的评价。

评价维度：

- 比例
- 颜色
- 场景
- 天气
- 风格一致性
- 单品质量
- 可立即调整动作

### 输出原则

- 必须给分，但分数不是重点。
- 必须告诉用户哪里好。
- 必须告诉用户怎么改。
- 不使用羞辱性词汇。
- 能识别出的单品应进入候选衣橱。

### 评价结果结构

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| total_score | int | 总分 |
| summary | string | 总评 |
| evidence | array | 分维度证据 |
| actions | array | 可执行调整 |
| closet_candidates | array | 候选入库单品 |
| next_tryon_prompt | string | 试穿图生成提示 |

## 15. 全局米粒聊天系统

### 产品定位

米粒聊天不是客服入口，而是大模型时代的产品主控层。用户可以在任意核心页面呼出米粒，用自然语言补充需求、解释偏好、调整生成结果。

第一阶段采用全局浮层：

- 首页：问今天怎么穿、切换场景、要求重新生成。
- 详情页：问为什么这样搭、哪里显瘦、能否换一件。
- 建档页：补充年龄、星座、城市、职业、常用场景、喜欢和不喜欢。
- 衣橱页：问某件衣服怎么搭。
- 点评页：上传照片后追问优缺点和修改方案。

### 信息采集原则

米粒聊天应把用户自然语言转成结构化偏好，但不能让用户感觉在填表。

示例：

```text
用户：我今天参加婚礼，不想抢新娘风头，也别太老气。
```

应解析为：

```json
{
  "scene": "wedding",
  "style_constraints": ["low_key", "elegant", "not_mature"],
  "avoid": ["too_white", "too_red", "too_shiny"]
}
```

### 快捷选项

聊天面板必须保留快捷按钮，降低输入成本：

- 更显瘦
- 换场景
- 用我的衣橱
- 别太成熟
- 找类似款

### 米粒形象

米粒形象原则：

- 不做强势美女，不制造压迫感。
- 普通、干净、亲近，但有审美判断。
- 头像优先使用简洁圆脸、短发或低马尾、低饱和服饰。
- 表情克制，避免虚拟偶像感。

## 17. 隐私、安全与审美边界

### 隐私原则

三视图、自拍、穿搭照都属于敏感图片，正式版本必须做到：

- 上传前明确授权。
- 告知用途：身材分析、试穿生成、衣橱识别。
- 提供删除入口。
- 使用短期签名 URL。
- 默认不公开展示用户图片。
- 未经明确同意，不用于模型训练。

### 审美安全

系统不能：

- 羞辱用户身材。
- 使用“胖、矮、腿短、缺陷”等打击性表达。
- 暗示用户必须变瘦、变白、变幼态。
- 把用户图片用于未经授权的营销素材。

推荐表达应改为：

- “优化比例”
- “拉高视觉重心”
- “减少压身高”
- “让肩线更清楚”
- “让内搭单穿也完整”

## 18. 商业模式

### 订阅制

月会员权益：

- 每日自动生成今日穿搭。
- 每天最多生成 10 张试穿效果图。
- 无限衣橱或更高衣橱容量。
- 高级场景推荐。
- AI偏好学习。
- 风格报告。

### 单次生成

免费用户权益：

- 每天最多生成 2 张试穿效果图。
- 生图额度用完后，可看激励视频广告额外解锁 1 张。
- 免费用户每天最多通过广告额外解锁 3 张，避免过度打扰。
- 可体验基础场景推荐。
- 可保存基础评分结果。

付费用户权益：

- 每天最多生成 10 张试穿效果图。
- 可切换更多场景。
- 可使用更完整的衣橱和趋势库推荐。
- 可获得更细的解释和风格学习。

单次生成适合：

- 新用户体验。
- 社交媒体引流。
- 特定场景，如婚礼、旅行、约会。

流程：

```text
上传三视图 / 自拍
→ 输入身高体重
→ 选择场景
→ 系统先查用户衣橱
→ 衣橱无法满足时调用趋势库或商品候选库
→ 生成试穿效果图
→ 用户评分或继续生成
```

### 商品推荐

商品推荐必须遵循：

- 优先用用户已有衣橱。
- 衣橱有可用单品时，不推荐购买。
- 衣橱缺口才进入趋势库或商品候选库。
- 用户没有衣橱时，可以用趋势库生成穿搭灵感，但不主动展示购买渠道。
- 用户主动询问购买渠道或点击“找类似款”时，才展示商品链接。
- 同类型衣服必须提供多个候选链接，不能只给一个。
- 标注商品来源。
- 广告和客观建议必须区分。

### 订阅入口

订阅入口分为三类：

1. 强触发入口：用户生图额度用完后弹窗提示。
2. 弱入口：我的页 / 个人页展示米粒 Plus 卡片。
3. 场景入口：高级能力触发时提示，如多套对比、连续风格计划、完整衣橱学习。

额度用完弹窗文案：

```text
标题：今天的免费试穿次数用完了
说明：米粒还能继续帮你搭。你可以看一段广告解锁 1 张，或开通米粒 Plus。
订阅文案：一杯咖啡的价格，让米粒陪你搭一个月。
按钮：看广告，解锁 1 张 / 开通米粒 Plus
```

### 激励视频广告

小程序端使用微信激励视频广告能力：

```text
wx.createRewardedVideoAd({ adUnitId })
```

前端处理原则：

- 只有广告完整播放结束，才增加 1 次生图额度。
- 用户中途关闭广告，不解锁额度。
- 广告加载失败时，不直接扣用户权益。
- 开发版允许 Mock 广告完成，正式版必须配置真实 `adUnitId`。
- 广告解锁和订阅权益必须由后端记录，不能只靠前端本地状态。

## 22. 当前原型设计决策

### 首页风格

采用杂志封面式首屏：

- 大图优先。
- 文字少。
- 结论直接放在图上。
- 场景选择放在首屏下方，避免挡住人脸。
- 主题选择放右上角。
- 天气与日期合并为顶部信息。
- 幸运色作为轻量胶囊信息。

### 主题

当前设计三种主题：

- 米色：日常、温柔、杂志感。
- 粉色：更女性化、更轻甜。
- 酷炫：更科技、更未来试衣镜。

### Logo

底部使用：

```text
竖排“闺蜜” + MILI.
```

定位为品牌水印，不应遮挡人物和内容，因此需保持一定透明度。

## 23. 核心指标

### 激活指标

- 完成三视图上传率。
- 首次试穿图生成完成率。
- 首次评分率。

### 留存指标

- 次日打开率。
- 连续 7 天查看今日穿搭人数。
- 场景切换次数。
- 收藏 / 点赞次数。

### 付费指标

- 免费用户生成后付费率。
- 会员开通率。
- 单次生成购买率。

### 产品质量指标

- “愿意现实穿”平均分。
- “像不像自己”平均分。
- 低分原因分布。
- 图片生成失败率。

## 24. 一句话总结

闺蜜的核心不是“生成一张好看的穿搭图”，而是让用户逐渐相信：

> 米粒知道我适合什么，也知道我今天要去哪里、天气怎么样、衣橱里有什么，并且能每天给我一套真的能穿出去的方案。
